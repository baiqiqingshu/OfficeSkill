# PPT Content Restructuring Strategies

当用户提供的内容过于密集时，agent 需要主动对内容进行重构以符合设计规范。本文档提供具体的重构策略。

---

## 1. 内容过多时的拆分策略

### 1.1 "总-分"拆分法

当单页内容超过 50 词或 5 条要点时：

**原始（不合规）：**
```
幻灯片：产品优势
• 高性能：采用最新架构，处理速度提升300%，支持并发1万连接
• 高可用：99.99%可用性保证，多区域部署，自动故障转移
• 安全性：端到端加密，SOC2认证，零信任架构
• 易用性：5分钟快速部署，无需运维，开箱即用
• 成本低：按需付费，无前期投入，节省60%费用
• 可扩展：水平扩展，支持10万QPS，弹性伸缩
• 生态丰富：200+集成，开放API，活跃社区
```

**重构后（合规）：**
```
幻灯片 1：产品核心优势（总览）
• 极致性能
• 企业级可靠性
• 银行级安全
• 开发者友好

幻灯片 2：性能与可靠性
• 处理速度提升 300%
• 99.99% 可用性 SLA
• 多区域自动容灾

幻灯片 3：安全与易用
• 端到端加密 + SOC2
• 5 分钟快速部署
• 按需付费，节省 60%
```

### 1.2 "时间线"拆分法

适用于流程、步骤类内容：

- 每 3-4 个步骤一页
- 用视觉连接符暗示页面间的连续关系
- 每页标题标注步骤范围（如 "步骤 1-3"、"步骤 4-6"）

### 1.3 "层次"拆分法

适用于分层结构内容：

```
第1页：高层架构图（概览）
第2页：某一层的详细说明
第3页：另一层的详细说明
```

## 2. 文本精简技巧

### 2.1 句子 → 关键词

| 原始文本 | 精简后 |
|:---|:---|
| 我们的系统采用了微服务架构来实现高可用性 | 微服务 → 高可用 |
| 通过自动化测试流水线确保代码质量 | 自动化测试保障质量 |
| 用户可以在5分钟内完成整个部署流程 | 5分钟极速部署 |

### 2.2 段落 → 结构化

将段落文字转化为：
- **图标 + 关键词** 组合
- **数字 + 简短描述** 组合
- **对比表格** 形式

### 2.3 数据 → 可视化

当幻灯片包含数字对比时：
- 3个以下数据点 → 大数字展示（数字 40pt+，说明 18pt）
- 4-7个数据点 → 柱状图/饼图
- 时间序列 → 折线图

## 3. 常见场景的重构模板

### 3.1 公司/产品介绍（10-15页标准结构）

```
1. 封面
2. 议程/目录
3. 公司概述（1页）
4. 问题/痛点（1-2页）
5. 解决方案（2-3页）
6. 产品特性/优势（2-3页）
7. 案例/数据（1-2页）
8. 团队/资质（1页）
9. 下一步/CTA（1页）
10. 结束页
```

### 3.2 技术方案汇报（12-20页标准结构）

```
1. 封面
2. 背景与目标
3. 现状分析（1-2页）
4. 方案概览（1页总览）
5. 技术架构（2-3页分层详述）
6. 实施计划/时间线（1-2页）
7. 风险与应对（1页）
8. 资源需求（1页）
9. 预期收益（1页）
10. Q&A / 结束
```

### 3.3 工作汇报/周报（8-12页标准结构）

```
1. 封面（标题+日期）
2. 本期摘要/亮点（关键数字）
3. 完成事项（2-3页）
4. 进行中事项（1-2页）
5. 问题与风险（1页）
6. 下期计划（1-2页）
7. 需要支持/决策事项（1页）
8. 结束
```

## 4. Agent 执行规则

### 4.1 自动拆分触发条件

当以下任一条件满足时，agent 必须主动拆分内容：

- 单页文本 > 50 词
- 单页要点 > 5 条
- 单条要点 > 15 词
- 需要在正文中使用 < 18pt 字体才能容纳内容
- 预计文本区域占比 > 40%

### 4.2 拆分时的提示词构建

拆分后在 officecli 的 prompt 中明确指出：

```
请为以下内容生成 PPT：
- 主题：[xxx]
- 总共 [N] 页
- 第 1 页：封面，标题"..."
- 第 2 页：内容概览，包含以下 4 个要点：...
- 第 3 页：详细说明第一个要点...
- ...

设计要求：
- 风格：专业商务 / 简约现代 / 创意活泼
- 每页不超过 5 条要点
- 正文字号不小于 20pt
- 保持充足留白
- 配色统一协调
```

### 4.3 不应拆分的情况

- 用户明确要求"放在一页"
- 内容已经符合密度规范
- 用户指定了精确的页数限制且内容无法再精简
  - 此时应精简文字而非拆分，但仍需保持最低字号要求
